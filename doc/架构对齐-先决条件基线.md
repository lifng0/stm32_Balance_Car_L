## 1. 模式枚举真值源

权威定义文件:[modes.yaml](../ros2_ws/src/balance_car_bringup/config/modes.yaml)

| 模式 ID | 模式名 | 允许的主要控制源 | 备注 |
| --- | --- | --- | --- |
| 0 | Normal | manual | 含蓝牙控制 |
| 1 | Weight_M | manual | 负重模式 |
| 3 | K210_Line | vision_line | K210 视觉循迹 |
| 4 | K210_Follow | vision_follow | K210 目标跟踪 |
| 7 | Lidar_Avoid | lidar_avoid | 雷达避障 |
| 8 | Lidar_Follow | lidar_follow / lidar_track | 雷达跟踪 |

### 必须保持一致的四处

| 位置 | 文件 | 当前状态 |
| --- | --- | --- |
| 固件枚举 | `USER/myenum.h` `Car_Mode` | ✅ 0/1/3/4/7/8 已定义 |
| 协调器 | `tools/pi_car_coordinator.py` `MODE_NAME` / `SUPPORTED_MODE_IDS` | ✅ 本次已补齐 7 |
| 串口桥 | `tools/pi_serial_bridge.py` `MODE_NAME` | ✅ 已含 0/1/3/4/7/8 |
| 控制仲裁 | `control_mux_node.py` `MODE_TO_SOURCES` | ✅ 改为从 modes.yaml 加载(回退表含 3/4/7/8) |

### 本次修复

- `pi_car_coordinator.py` `SUPPORTED_MODE_IDS` 由 `{0,1,3,4,8}` 修正为 `{0,1,3,4,7,8}`。
  此前缺 7 导致后端 `CMD_SET_MODE` 拒绝设入 Lidar_Avoid,而 `lidar_avoid_node` 与
  `control_mux_node` 都按 `mode==7` 工作 —— 避障模式实际无法通过后端激活。

### 待评审缺口(未改动,属固件行为变更)

要让 Lidar_Avoid(7) 真正可用,除协调器外还需改固件并重新烧录:

1. `APP/mode/app_mode.c` 的 `Mode_Next/Mode_Previous` 轮换与 `Mode_IsSelectable`
   **不包含 Lidar_Avoid**,拧轮 UI 无法选中避障。
2. `BSP/bsp_pi_comm.c` 的 `PI_Comm_IsModeSupported` **不含 7**,固件直接 NACK 掉 `CMD_SET_MODE(7)`。
3. `BSP/bsp_pi_comm.c` 的 `CMD_SET_MOVE` 门控条件**不含 Lidar_Avoid**,避障模式下运动命令被 NACK。

→ 当前 Lidar_Avoid 在固件层是「选不到 / 设不进 / 动不了」三重死锁,协调器补 7 只是必要不充分条件。

## 2. 设备路径配置真值源

权威定义文件:[devices.yaml](../ros2_ws/src/balance_car_bringup/config/devices.yaml)

当前真值取自 `tools/pi_car_coordinator.service`:

| 设备 | 路径 | 波特率 |
| --- | --- | --- |
| STM32 | `/dev/serial/by-path/platform-xhci-hcd.1-usb-0:1:1.0-port0` | 921600 |
| K210 | `/dev/serial/by-path/platform-xhci-hcd.1-usb-0:2:1.0-port0` | 115200 |
| 雷达 (CP2102) | `/dev/serial/by-id/usb-Silicon_Labs_CP2102_USB_to_UART_Bridge_Controller_0001-if00-port0` | 230400 |

### 现状与目标

- **现状**:`system.launch.py` 已读取 `devices.yaml` 的雷达段传给 `tminiplus_node`(本次接线);
  STM32/K210 路径仍硬编码在 `pi_car_coordinator.service`,待第 2/3 阶段对应 ROS 节点建立后再收敛。
- **目标**:后续 service / launch / compose / 各硬件节点统一引用 `devices.yaml`,
  不再各自硬编码(总方案 12.7)。
- **下沉到容器内节点时必须复核**:`dialout` 附加组、`/dev/serial/by-id/*`、`by-path/*` 权限。
