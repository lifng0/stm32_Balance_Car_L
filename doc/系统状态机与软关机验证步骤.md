# 系统状态机与软关机验证步骤

更新时间：2026-06-02

本文用于验证以下流程已经打通：

- 上电后 `STM32` 等待树莓派与雷达初始化
- `OLED` 实时显示 `PI / LIDAR / SYSTEM` 初始化状态
- 树莓派准备完成后进入模式选择
- 选好模式后提示 `press key to start`
- 按下 `Key1` 后，`STM32` 向树莓派发送开始事件
- 树莓派查询当前模式并进入执行态
- 执行期间实时接收 `Stop_Flag` 变化
- 再按一次 `Key1` 返回模式选择
- 长按 `Key1` 触发树莓派软关机

## 1. 预备条件

- 小车已烧录最新 `STM32` 固件
- 树莓派已部署并启用：
  - `~/workspace/balance_car/scripts/pi_serial_bridge.py`
  - `~/workspace/balance_car/scripts/tminiplus_bridge.py`
  - `~/workspace/balance_car/scripts/pi_car_coordinator.py`
  - `balance-car-coordinator.service`
- `STM32 -> TTL转USB -> 树莓派`
- `T-mini Plus -> 官方转接板 -> 树莓派`

## 2. 树莓派服务检查

```bash
systemctl status balance-car-coordinator.service --no-pager
journalctl -u balance-car-coordinator.service -n 50 --no-pager
```

期望现象：

- 服务状态为 `active (running)`
- 日志中能看到：
  - `reported PI_READY`
  - `lidar ready ...`
  - `reported SYSTEM_READY`

## 3. 上电初始化验证

1. 先给树莓派上电，再给整车系统上电。
2. 观察 `OLED`。

期望现象：

- 上电初期显示：
  - `Pi system init...`
  - `PI: WAIT/READY`
  - `LIDAR: WAIT/READY`
  - `SYSTEM: BOOTING/READY`
- 树莓派与雷达初始化完成后，系统自动进入模式选择界面。

## 4. 模式选择与开始验证

1. 通过转动车轮切换模式。
2. 选定模式后按一次 `Key1`。

期望现象：

- `OLED` 先显示 `press key to start`
- 按下 `Key1` 后，`OLED` 显示 `running...`
- 树莓派日志中可看到：

```text
[pi-coordinator] event=START_REQUEST
[pi-coordinator] start mode=...
```

## 5. 被拿起暂停验证

1. 小车进入运行态后，将小车拿起。
2. 再将小车放回。

期望现象：

- 被拿起时，`STM32` 内部 `Stop_Flag` 置位
- `OLED` 可看到 `paused`
- 树莓派日志中依次出现：

```text
[pi-coordinator] event=STOP_ASSERT
[pi-coordinator] pause by pickup
```

- 放回后日志出现：

```text
[pi-coordinator] event=STOP_CLEAR
[pi-coordinator] pickup cleared
```

## 6. 返回模式选择验证

1. 运行中短按一次 `Key1`。

期望现象：

- 系统停止当前执行
- 返回模式选择界面
- 树莓派日志中出现：

```text
[pi-coordinator] event=MODE_SELECT
[pi-coordinator] return to mode select
```

## 7. 长按关机验证

1. 任意状态下长按 `Key1` 超过约 `2s`。

期望现象：

- `OLED` 显示：
  - `shutdown request`
  - `wait pi poweroff`
- 树莓派日志中出现：

```text
[pi-coordinator] event=SHUTDOWN_REQUEST
[pi-coordinator] shutdown requested by stm32
```

- 随后 `OLED` 显示：
  - `pi shutdown ok`
  - `now cut main power`

看到这两行之后，再关闭总电源。

## 8. 备注

- 不建议频繁直接断电关闭树莓派，这会增加文件系统损坏和存储介质异常的风险。
- 后续如果要进入真正的模式执行逻辑，可以直接在 `pi_car_coordinator.py` 的 `START_REQUEST` 分支里继续接入具体任务代码。
