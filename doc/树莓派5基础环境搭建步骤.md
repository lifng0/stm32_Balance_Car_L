# 树莓派5基础环境搭建步骤

更新时间：2026-06-02

## 1. 本阶段目标

本阶段只解决三件事：

- 让树莓派5在 `Raspberry Pi OS 64-bit` 下稳定启动并供电可靠
- 建好后续开发所需的 `Raspberry Pi OS + Docker` 基础环境
- 为后续 `STM32` 串口联调和 `T-mini Plus` 雷达接入预留好系统条件

本阶段不做：

- 不接入 `STM32` 正式协议
- 不接入 `T-mini Plus` 正式驱动
- 不做建图、避障、跟随等上层功能
- 不在宿主机上强行走 `Ubuntu` 专用的 `ROS 2 Jazzy` 二进制安装路径

## 2. 推荐基线

截至 `2026-06-02`，建议采用下面这套最稳妥的基线：

- 硬件：`Raspberry Pi 5 8GB`
- 系统：`Raspberry Pi OS 64-bit`
- 宿主机职责：设备接入、网络、串口、Docker 运行环境
- ROS 2 路线：优先走 `Docker` 容器中的 `ROS 2 Jazzy`
- 容器：官方 `Docker Engine`

这样选的原因：

- `Raspberry Pi OS` 是官方推荐系统，适合当前树莓派5硬件
- `Raspberry Pi OS` 基于 Debian，而 `ROS 2 Jazzy` 官方二进制主支持目标是 `Ubuntu 24.04`
- `ROS 2` 官方关于树莓派的说明里明确提到：`Raspberry Pi OS` 属于 Debian 路线，适合通过容器方案运行 `ROS 2`
- 对我们当前“尽快稳定交付”的目标来说，宿主机专注设备管理，`ROS 2` 放进容器，后续最省事

## 3. 本阶段交付物

完成后应达到以下状态：

- 树莓派5可稳定上电运行
- 可通过网络 `SSH` 登录
- 系统时区、主机名、普通用户、基础工具齐全
- `Docker` 可正常运行
- 已创建后续项目工作目录
- 串口设备识别和权限策略已准备好
- 已具备后续运行 `ROS 2` 容器的条件

## 4. 实施步骤

## 4.1 步骤0：确认供电与存储方案

建议先冻结这两个硬件前提：

- 树莓派5供电走 `电池 -> 降压稳压模块 -> 树莓派5`
- 启动盘优先使用质量可靠的 `microSD` 或 `NVMe`

执行要求：

- 降压模块输出先空载调到稳定 `5V`
- 后续上树莓派之前，再带载复测一次输出是否仍稳定
- 如果后面接入雷达和多个 `USB` 外设，供电能力需要预留余量

验收：

- 上电后树莓派不反复重启
- 空闲和联网状态下无欠压异常

## 4.2 步骤1：刷写系统

推荐安装：

- `Raspberry Pi OS 64-bit`

建议做法：

1. 用 `Raspberry Pi Imager` 刷写 `Raspberry Pi OS 64-bit`。
2. 首次就启用以下预配置：
   - 主机名
   - 用户名和密码
   - `SSH`
   - 网络
   - 时区
3. 如果你准备用无显示器方式启动，优先一次性把网络和 `SSH` 配好。

建议命名：

- 主机名：`pi5-car`

验收：

- 树莓派首次启动成功
- 能在局域网中找到设备
- 能用 `SSH` 登录

## 4.3 步骤2：系统初始化

登录树莓派后先做基础更新：

```bash
sudo apt update
sudo apt full-upgrade -y
sudo reboot
```

重启后安装常用工具：

```bash
sudo apt install -y \
  git curl wget vim tmux htop net-tools \
  ca-certificates gnupg lsb-release \
  build-essential python3-pip python3-venv \
  minicom screen usbutils
```

设置时区：

```bash
timedatectl
sudo timedatectl set-timezone Asia/Shanghai
```

建议补充：

- 开启 `SSH` 开机自启
- 固定树莓派局域网地址或在路由器里绑定 DHCP

验收：

- `sudo apt update` 无报错
- `timedatectl` 显示时区为 `Asia/Shanghai`
- 常用命令可正常使用

## 4.4 步骤3：准备串口与 USB 设备基础能力

因为后面：

- `STM32` 将通过 `TTL转USB` 接入树莓派 `USB`
- `T-mini Plus` 也将通过 `USB转UART` 接入树莓派

所以这一阶段先把串口环境准备好。

建议检查：

```bash
ls /dev/ttyUSB*
ls /dev/serial/by-id/
```

说明：

- 当前还没正式接设备时，这两个命令可能为空
- 真正目的是确认后面系统能用稳定路径识别 `USB` 串口设备

验收：

- 系统已具备串口排查工具
- 了解后续优先使用 `/dev/serial/by-id/`，不写死 `/dev/ttyUSB0`

## 4.5 步骤4：安装 Docker Engine

`Raspberry Pi OS` 属于 Debian 路线，这里按 Debian 官方方式安装 `Docker Engine`。

先卸载可能冲突的旧包：

```bash
for pkg in docker.io docker-doc docker-compose podman-docker containerd runc; do
  sudo apt-get remove -y $pkg
done
```

添加官方仓库：

```bash
sudo apt-get update
sudo apt-get install -y ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/debian/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc

echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/debian \
  $(. /etc/os-release && echo \"$VERSION_CODENAME\") stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt-get update
```

安装 Docker：

```bash
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

把当前用户加入 `docker` 用户组：

```bash
sudo usermod -aG docker $USER
newgrp docker
```

验证：

```bash
docker --version
docker compose version
docker run hello-world
```

验收：

- `docker --version` 正常返回版本
- `docker run hello-world` 成功

## 4.6 步骤5：确认 ROS 2 路线

这一版系统下，不建议在宿主机上照搬 `Ubuntu 24.04` 的 `ROS 2 Jazzy apt` 安装方法。

本项目在 `Raspberry Pi OS 64-bit` 下采用下面的原则：

- 宿主机先不强装 `ROS 2 Jazzy` 二进制环境
- 先把 `Docker` 环境稳定下来
- 后续 `ROS 2` 相关节点优先放进容器运行

这一阶段你只需要确认下面结论：

- 树莓派宿主机负责：网络、串口、USB、日志、容器运行
- `ROS 2` 负责的内容：后续放在容器中处理

验收：

- 团队内部已经统一这个路线
- 不再在宿主机上尝试 `Ubuntu` 专用的 `ros-jazzy-*` 安装命令

## 4.7 步骤6：创建项目工作目录

建议目录结构如下：

```bash
mkdir -p ~/workspace/balance_car
mkdir -p ~/workspace/balance_car/docker
mkdir -p ~/workspace/balance_car/ws/src
mkdir -p ~/workspace/balance_car/scripts
mkdir -p ~/workspace/balance_car/logs
mkdir -p ~/workspace/balance_car/services
```

说明：

- `docker` 目录：后续放 `Dockerfile`、`compose.yaml`
- `ws` 目录：后续容器内共享的 ROS 工作区
- `scripts` 目录：放部署脚本、串口排查脚本
- `logs` 目录：放运行日志

验收：

- 目录创建完成
- 工作区路径后续可直接复用

## 4.8 步骤7：做阶段验收

这一阶段完成后，请至少逐项确认下面这些结果：

```bash
uname -a
cat /etc/os-release
docker --version
docker compose version
timedatectl
hostnamectl
```

同时检查：

- 能否通过 `SSH` 稳定登录
- 断电重上后能否自动恢复网络
- 长时间空载运行是否有重启或掉线
- 接入 `TTL转USB` 或雷达转接板后，系统是否能识别到新的 `USB` 串口

## 5. 建议的实际执行顺序

为了少走弯路，建议你按这个顺序做：

1. 先搞定树莓派供电稳定
2. 再刷 `Raspberry Pi OS 64-bit`
3. 再做系统更新和常用工具安装
4. 再装 `Docker`
5. 再确认串口工具和 `USB` 识别链路
6. 最后创建工作目录并做验收

## 6. 本阶段常见风险

### 风险1：供电不稳

表现：

- 随机重启
- USB 串口设备频繁掉线
- 网络偶发断开

处理：

- 先查降压模块输出
- 再查线材压降
- 再查 USB 外设负载

### 风险2：误走 Ubuntu 专用 ROS 2 安装路径

表现：

- `ros-jazzy-*` 包安装过程报兼容性问题
- 依赖链开始混乱

处理：

- 当前系统固定为 `Raspberry Pi OS 64-bit`
- 本阶段不在宿主机强行装 `ROS 2 Jazzy apt` 包
- `ROS 2` 后续优先容器化

### 风险3：串口设备号漂移

表现：

- 重插设备后 `/dev/ttyUSB0` 和 `/dev/ttyUSB1` 对调

处理：

- 后续统一使用 `/dev/serial/by-id/`
- 必要时再补充 `udev` 规则
