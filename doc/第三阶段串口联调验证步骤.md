# 第三阶段串口联调验证步骤

更新时间：2026-06-02

## 1. 当前已完成项

在真正插上 `TTL转USB` 模块前，以下内容已经完成：

- `STM32` 工程已加入 `USART3` 通讯协议代码
- 协议支持：
  - `PING`
  - `SET_ENABLE`
  - `SET_MODE`
  - `SET_MOVE`
  - `QUERY_STATUS`
  - `HEARTBEAT`
  - `EMERGENCY_STOP`
  - `ACK`
  - `NACK`
  - `STATUS`
  - `EVENT`
- `STM32` 本地编译通过
- 树莓派脚本 `~/workspace/balance_car/scripts/pi_serial_bridge.py` 已部署完成
- 树莓派脚本依赖 `python3-serial` 已确认可用

## 2. 接线前提

接线方式默认如下：

- `STM32 UART3 TX -> TTL转USB RX`
- `STM32 UART3 RX -> TTL转USB TX`
- `STM32 GND -> TTL转USB GND`
- `TTL转USB USB -> 树莓派 USB`

注意：

- 不需要给 `TTL转USB` 反向供电
- 树莓派继续由独立降压模块供电

## 3. 上电后第一步：确认树莓派识别到了串口

在树莓派上执行：

```bash
ls /dev/serial/by-id/
ls /dev/ttyUSB*
```

验收标准：

- 能看到新的 `USB` 串口设备
- 优先记录 `/dev/serial/by-id/xxx` 这种稳定路径

## 4. 第二步：确认 STM32 已烧录新固件

烧录当前工程最新固件后，上电进入正常模式。

建议先满足：

- 小车本体能正常起立
- 没有因为新协议代码导致原有功能异常

## 5. 第三步：做最小连通测试

在树莓派上执行：

```bash
python3 ~/workspace/balance_car/scripts/pi_serial_bridge.py \
  --device /dev/serial/by-id/你的设备路径 \
  ping
```

验收标准：

- 能收到类似：

```text
ACK cmd=0x01 seq=1
```

如果没有响应，优先检查：

- `TX/RX` 是否接反
- `GND` 是否共地
- 波特率是否都是 `115200`
- `STM32` 是否已烧录新固件

## 6. 第四步：查询状态

执行：

```bash
python3 ~/workspace/balance_car/scripts/pi_serial_bridge.py \
  --device /dev/serial/by-id/你的设备路径 \
  status
```

验收标准：

- 能收到类似：

```text
STATUS mode=Normal stop=1 low_power=0 move_x=0.0 move_z=0.0 battery=12.00 angle=...
```

## 7. 第五步：启停控制测试

先使能：

```bash
python3 ~/workspace/balance_car/scripts/pi_serial_bridge.py \
  --device /dev/serial/by-id/你的设备路径 \
  enable 1
```

再急停：

```bash
python3 ~/workspace/balance_car/scripts/pi_serial_bridge.py \
  --device /dev/serial/by-id/你的设备路径 \
  stop
```

验收标准：

- 都能收到 `ACK`
- `stop` 后再查询状态时，`stop=1`

## 8. 第六步：模式切换测试

先切到普通模式：

```bash
python3 ~/workspace/balance_car/scripts/pi_serial_bridge.py \
  --device /dev/serial/by-id/你的设备路径 \
  mode 0
```

再查询：

```bash
python3 ~/workspace/balance_car/scripts/pi_serial_bridge.py \
  --device /dev/serial/by-id/你的设备路径 \
  status
```

验收标准：

- 收到 `ACK`
- 状态中的 `mode=Normal`

## 9. 第七步：速度命令测试

在普通模式下执行：

```bash
python3 ~/workspace/balance_car/scripts/pi_serial_bridge.py \
  --device /dev/serial/by-id/你的设备路径 \
  move 5 0
```

然后查询状态：

```bash
python3 ~/workspace/balance_car/scripts/pi_serial_bridge.py \
  --device /dev/serial/by-id/你的设备路径 \
  status
```

验收标准：

- `move` 命令收到 `ACK`
- `STATUS` 中看到 `move_x=5.0`

注意：

- 初次测试建议把速度设得很小
- 建议先架空轮子或做防护，避免直接冲车

## 10. 第八步：心跳与超时保护测试

在树莓派上执行：

```bash
python3 ~/workspace/balance_car/scripts/pi_serial_bridge.py \
  --device /dev/serial/by-id/你的设备路径 \
  heartbeat --count 20 --interval 0.1
```

验收标准：

- 周期性看到：

```text
HEARTBEAT_ACK seq=...
```

然后停止发送心跳，等待约 `0.5s` 后再次查状态：

```bash
python3 ~/workspace/balance_car/scripts/pi_serial_bridge.py \
  --device /dev/serial/by-id/你的设备路径 \
  status
```

期望结果：

- `STM32` 已进入安全停止
- `stop=1`
- `Move_X / Move_Z` 被清零

## 11. 验证顺序建议

建议严格按下面顺序测：

1. 串口设备是否识别
2. `PING`
3. `STATUS`
4. `ENABLE`
5. `MODE`
6. `MOVE`
7. `HEARTBEAT`
8. 超时保护

## 12. 当前已知限制

当前这版实现的实际限制如下：

- `SET_MOVE` 只允许在 `Normal` 或 `Weight_M` 下使用
- 目前默认复用了现有模式体系，没有单独新增 `Pi_Control` 模式
- 还没有做 `ROS 2` 节点版本，目前先是 Python 串口桥脚本
